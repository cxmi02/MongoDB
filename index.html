Investigacion Views y MongoDB.

1. Investigar qué son las VIEWS en MYSQL.

Una VIEW es una tabla virtual con su conjunto de filas y columnas, cuyo contenido está definido por una consulta SELECT sobre una o más tablas. Esto quiere decir, que los datos que contienen noestán almacenados realmente en la vista, si no que se generan de manera dinámica de acuerdo a la consulta que ejecuten. Las VIEW, se utilizan a menudo como filtros de las tablas de las que recogen datos.

Tipos de vistas

  1. Vistas de usuario: El contenido esta definido por el usuario por medio de una consulta.
  2. Vistas indexadas: Los datos se almacenan en una tabla real.
  3. Vistas con particiones: Combinan datos de uno o más servidores.
  4. Vistas del sistema: Consultas de metados.

2. Investigar que son los procedimientos almacenados.

Un procedimiento almacenado MySQL no es más que una porción de código que puedes guardar y reutilizar, Para poder crear un procedimiento almacenado es necesario que tengas permisos INSERT y DELETE sobre la base de datos.

3. Cual es la diferencia entre SQL y NoSQL.

  • Las SQL proporcionan una capacidad de escalar baja, en comparación con las NoSQL. Esta es una de las principales ventajas de las NoSQL, ya que están pensadas para grandes volúmenes de información como el Big Data. Lo anterior es debido a que las SQL están centralizadas y las NoSQL distribuidas, posibilitando que se ejecuten en múltiples máquinas pero con muy pocos recursos (RAM, CPU, disco…).
  • Las SQL se utilizan en múltiples aplicaciones de todo tipo, las NoSQL se emplean principalmente para el Big Data (por ejemplo en redes sociales).
  • La rapidez de ambas BBDD va a depender del contexto o de su uso: en datos estructurados las SQL son más rápidas, pero como vimos anteriormente, el Big Data no es estructurado y es ahí donde consiguen mucha mayor rapidez las NoSQL.

4. Investigar qué es MongoDB.

es un sistema de gestión de bases de datos (DBMS) no relacional de código abierto que utiliza documentos flexibles en lugar de tablas y filas para procesar y almacenar diversas formas de datos. MongoDB no requiere un sistema de gestión de bases de datos relacionales (RDBMS), por lo que proporciona un modelo de almacenamiento de datos flexible que permite a los usuarios almacenar y consultar tipos de datos multivariados con facilidad.
